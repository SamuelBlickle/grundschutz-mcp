# Security Policy

This is a security-adjacent tool. Please report vulnerabilities responsibly.

## Reporting

Do not open a public issue for security problems. Use GitHub's private
"Report a vulnerability" feature, or contact the maintainer directly via the
contact on https://samuelblickle.de.

## Supply chain

- Dependencies are pinned.
- Releases follow SemVer and are signed.
- The BSI data source is pinned to a reviewed commit; updates are deliberate.
